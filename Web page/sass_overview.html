Обзор Sass

Введение

SASS - это препроцессор, который позволяет использовать функции недоступные в самом CSS, например, переменные, вложенности, миксины, наследование и т.д. Язык SASS имеет два синтаксиса:

  1. SASS (Syntactically Awesome Style Sheets) - отличается отсутствием фигурных скобок, в нём вложенные элементы реализованы с помощью отступов;
  2. SCSS (Sassy CSS) - использует фигурные скобки, как и сам CSS.

Согласно официальному сайту существует несколько способов установки SASS - посредством приложения (CodeKit, Hammer, Prepros и т.д.), библиотек, либо командной строки с использованием менеджера пакетов npm, входящего в Node.js, при выполнении команды:

npm install -g sass

При этом браузер не воспринимает код SASS. Поэтому и требуется препроцессор для преобразования кода SASS в стандартный CSS. Этот процесс называется транспилированием. Можно использовать программу Koala, которая позволяет быстро и легко компилировать SASS, используя графический интерфейс:

Переменные

Переменные SASS предлагают способ хранения информации, которую можно повторно использовать позже. SASS позволяет хранить информацию следующих типов:

  1. Строка (string);
  2. Число (number);
  3. Цвет (color);
  4. Логическое значение (boolean);
  5. Список (list);
  6. Null.

В Sass используется символ $, за которым следует имя, необходимое для объявления переменной. Например:

$variablename: value;

Переменные SASS доступны только на том уровне вложенности, где они определены.

Поведение по умолчанию для области видимости переменной можно переопределить с помощью метки !global. Метка !global указывает, что переменная является глобальной, т.е. она доступна на всех уровнях.

Вложенность

SASS позволяет вкладывать селекторы CSS таким же образом, как HTML. За счет этого код выглядит чище и становится более читабельным, по сравнению со стандартным CSS.

Многие свойства CSS имеют один и тот же префикc (к примеру, font-family, font-size и font-weight). С использованием SASS становится возможным записать эти свойства как вложенные. Транспилятор SASS преобразует их в обычный CSS.

Фрагментирование и импорт

При использовании SASS существует возможность создавать фрагменты SASS-файла, которые будут содержать в себе небольшие части CSS, которые впоследствии можно будет использовать в других SASS-файлах. Это позволяет сделать CSS модульным, что, в свою очередь, облегчает его обслуживание. Фрагмент - это Sass-файл, имя которого начинается с нижнего подчеркивания, например, _partial.scss. Нижнее подчеркивание в имени SASS-файла подразумевает, что это только фрагмент и он не должен транспилироваться в CSS. Фрагменты SASS подключаются при помощи директивы @import:

@import filename;

Указывать расширение файла не требуется, т.к. SASS автоматически предполагает, что подразумевается файл с расширением .sass или .scss. Также присутствует возможность импортировать файлы CSS. При этом любые переменные или миксины, определенные в импортированном файле, могут затем использоваться в основном файле.

Миксины

Директива @mixin используется при объявлении миксина и позволяет создавать код CSS, который будет повторно использоваться на веб-сайте:

@mixin name {
  property: value;
  property: value;
  ...
 }

Директива @include необходима для использования (включать) миксин.

Миксин также может включать в себя другие миксины и принимать аргументы. Помимо этого, возможно определить значения по умолчанию для переменных миксина.

Наследование

Директива @extend позволяет передавать набор свойств CSS от одного селектора к другому.

.selector2 {
  @extend .selector1;
  ...
 }

Функции

В SASS, как и в стандартном CSS, доступны операции сложения, вычитания, умножения и деления, однако необязательно использовать функцию calc(). Также в SASS работают операторы сравнения (==, !=, <, > и т.д.) и логические операторы (and, or, not).

Для проверки условий присутствуют директивы @if и @else. Директива @for используется для повторения выражений несколько раз подряд.

random() Возвращает случайное число от 0 до 1
